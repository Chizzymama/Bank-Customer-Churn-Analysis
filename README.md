# Bank-Customer-Churn-Analysis
This project analyzes customer churn in a Multinational Bank between (2018 -2022) dataset using Exploratory Data Analysis (EDA), Machine Learning, and Tableau. The goal is to identify key drivers of churn and develop predictive models to support customer retention strategies.

Dataset
- Customers: 10,000
- Features: 21
- Target Variable: Exited (Churn)
- Churn Rate: 20.38%

Project Workflow
1.	Data Cleaning and Preparation
2.	Exploratory Data Analysis (EDA)
3.	Feature Engineering
4.	Machine Learning Modeling
5.	Tableau Dashboard Development
6.	Business Recommendations

Key Findings
-	Complaint status was the strongest indicator of churn.
-	Customers aged 50–60 had the highest churn risk.
-	Inactive customers were more likely to leave the bank.
-	Customers with multiple products showed stronger retention.


Model Performance: The best practical model was Random Forest (Without Complaint):
-	Accuracy: 86.80%
-	Precision: 79.51%
-	Recall: 47.55%
-	F1 Score: 59.51%

Dashboard: The Tableau dashboard provides interactive insights into:
-	Churn by Age Group
-	Churn by Complaint Status
-	Churn by Number of Products
-	Churn by Activity Status


Recommendations
-	Improve complaint resolution processes.
-	Increase engagement among inactive customers.
-	Focus retention efforts on older customers.
-	Encourage customers to adopt multiple banking products.
